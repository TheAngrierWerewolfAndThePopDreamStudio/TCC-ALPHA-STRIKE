Web Development



Estratégia de Desenvolvimento

  No processo de construção do portal Alpha Strike, nossa prioridade é garantir que ele seja estável e seguro. Optamos por uma abordagem que nos permite manter o sistema de forma fácil e adicionar novos recursos conforme necessário, para acompanhar as mudanças na comunidade de airsoft ao longo do tempo. Para isso, escolhemos tecnologias modernas que garantem que o portal seja rápido e responsivo. Também tomamos medidas para proteger as informações dos usuários e manter o sistema seguro. Além disso, adotamos uma forma de desenvolvimento ágil, que nos permite fazer ajustes rapidamente e de forma colaborativa.

Objetivos:

  • Criar uma experiência de usuário intuitiva e envolvente, atendendo às necessidades e expectativas dos jogadores de airsoft.
  • Oferecer informações detalhadas sobre o esporte, incluindo regras, estrutura de jogos, equipamentos necessários e dicas de segurança.
  • Disponibilizar uma variedade de conteúdos multimídia, como artigos, vídeos e notícias, para entreter e educar os visitantes do site.
  • Facilitar a compra de equipamentos e acessórios de airsoft por meio de uma loja virtual integrada, proporcionando uma experiência de compra segura e conveniente.
  • Permitir o agendamento online de jogos e treinamentos, simplificando o processo de reserva e comunicação para os participantes.

Contexto do Projeto:

  O projeto é desenvolvido para o cliente Alpha Strike, uma empresa dedicada à promoção e prática do airsoft. Sob a liderança conjunta do The Angrier Werewolf e do Pop Dream Studio, busca-se expandir a presença online da Alpha Strike, oferecendo uma plataforma digital que complemente suas atividades presenciais e forneça recursos adicionais para sua comunidade de jogadores.

Tecnologias Utilizadas

  Para o desenvolvimento do site ou aplicativo, foram utilizadas as seguintes tecnologias:

Linguagens:

  • HTML: Utilizado para a estruturação e marcação do conteúdo das páginas web.
  • CSS: Utilizado para estilizar e formatar o layout das páginas web.
  • JavaScript: Utilizado para adicionar interatividade e dinamismo às páginas web.
  • Python: Utilizado para a programação do backend do site, incluindo integração com o banco de dados.
  • SQL: Utilizado para a manipulação e consulta de dados no banco de dados.

Frameworks e Bibliotecas:

  • Flask: Framework web em Python utilizado para desenvolver a aplicação web do lado do servidor.
  • Pyodbc: Biblioteca em Python utilizada para a conexão e interação com o banco de dados SQL Server.
  • Bootstrap: Framework front-end utilizado para facilitar o desenvolvimento de interfaces responsivas e atrativas.
  • Hashlib: Biblioteca em Python utilizada para implementar algoritmos de hash, como MD5, SHA-1 e SHA-256, que são comumente usados para garantir a integridade e segurança dos dados através da geração de resumos criptográficos.
  • Binascii: Biblioteca em Python que fornece funções para conversão entre representações binárias e ASCII, frequentemente usada para a manipulação e transformação de dados binários em formatos legíveis ou transferíveis, como a codificação em hexadecimal ou base64.

Software e Ferramentas:

  • Visual Studio Code: Editor de código utilizado para escrever e editar o código fonte do projeto.
  • SQL Server Management Studio: Ferramenta de gerenciamento e administração do banco de dados SQL Server.
  • Figma: Ferramenta de design colaborativo utilizada para criar wireframes, protótipos e designs de interface de usuário.
  • Visily.Ai: Plataforma de design de interfaces de usuário
  • Playground.Ai: Plataforma que permite aos usuários criar e explorar arte gerada por inteligência artificial, utilizando descrições de texto para gerar imagens digitais.
  • Canva: lataforma de design gráfico utilizada para criar elementos visuais como gráficos, banners e ícones.
  • Zed Attack Proxy (ZAP): Ferramenta de teste de segurança utilizada para identificar e corrigir potenciais vulnerabilidades de segurança.

Extensões:

  • Colorful Comments/Better Comments: Extensão para adicionar cores, melhorar a legibilidade e organização dos comentários no código fonte.
  • Indent-Rainbow: Extensão para colorir o recuo do código para facilitar a identificação de blocos de código.
  • Live Preview: Extensão para visualizar as alterações em tempo real enquanto se edita o código.

  Essas tecnologias foram escolhidas para proporcionar uma experiência de desenvolvimento eficiente e garantir um resultado final de alta qualidade para o portal Alpha Strike.

Estratégia de Armazenamento e Recuperação de Dados

 A estratégia de armazenamento e recuperação de dados do site é projetada para garantir a segurança, integridade e disponibilidade das informações essenciais. Aqui estão os principais pontos da nossa abordagem:

  1. Backups Incrementais e Completos

    • Backups Incrementais: Todos os dados são copiados e armazenados imediatamente após cada alteração realizada por usuários ou administradores. Isso inclui modificações em dados pessoais dos usuários, detalhes de produtos e agendamentos. Esses backups incrementais garantem que versões específicas dos dados possam ser recuperadas, se necessário.
    • Backup Completo Semanal: Além dos backups incrementais, um backup completo de todos os dados do site é realizado semanalmente. Esse backup abrange todo o banco de dados, assegurando uma camada adicional de proteção e a capacidade de restaurar o site a um estado recente em caso de falhas ou emergências.

  2. Procedimentos de Recuperação

    • Estabeleceremos procedimentos claros e documentados para a recuperação de dados conforme necessário. Em caso de perda de dados devido a falhas técnicas, ataques cibernéticos ou outros incidentes, nossa equipe fará um plano de ação para restaurar os dados a partir dos backups mais recentes disponíveis.
    • A recuperação de dados pode incluir a restauração de backups completos ou incrementais, dependendo da natureza e extensão do incidente. Esses procedimentos devem ser periodicamente revisados e testados para garantir sua eficácia e a capacidade de resposta rápida diante de crises.

  3. Segurança e Integridade dos Dados

    • Todos os backups deverão ser armazenados de forma segura em locais específicos, protegidos por medidas de segurança cibernética. Isso inclui criptografia de dados, acesso restrito baseado em permissões e monitoramento contínuo para detectar e mitigar potenciais vulnerabilidades.
    • A integridade dos backups deve ser verificada regularmente para garantir que todas as cópias estejam completas e livres de corrupção, proporcionando confiança na capacidade de restaurar dados precisos e completos em momentos críticos.

 Essa abordagem para armazenamento e recuperação de dados não apenas protege informações sensíveis dos usuários e operações essenciais do site, mas também assegura que possa ser mantida a continuidade dos serviços com o mínimo de interrupção. A constante avaliação e aprimoramento dos processos garantem que estamos preparados para enfrentar desafios futuros e proteger os interesses de todos os envolvidos com o site.

APIs e Integrações

Descrição das APIs Utilizadas

  1. Integração com Banco de Dados e Autenticação de Usuários:

    • Descrição: Implementamos APIs internas para integrar com o banco de dados, permitindo a realização de operações CRUD (Create, Read, Update, Delete) necessárias para funcionalidades de login, cadastro e formulários de contato.
    • Segurança: Utilizamos técnicas de hashing e salting para garantir a segurança das senhas dos usuários, armazenando-as de forma segura no banco de dados.
    • Fluxo:
      • Cadastro:Durante o cadastro, a senha do usuário é criptografada e o salt gerado é armazenado no banco de dados junto com a senha criptografada.
      • Login:Durante o login, a senha fornecida pelo usuário é comparada com a senha armazenada no banco de dados após ser criptografada usando o mesmo salt.

  2. Integração ao Sistema:

    • Método de Integração: A integração é realizada através de chamadas HTTP, onde os endpoints definidos no backend respondem às requisições do frontend. As operações de cadastro e login são processadas por meio dessas chamadas, garantindo a comunicação entre as diferentes camadas do sistema.
    • Modularidade: No backend, são desenvolvidos módulos de serviço que encapsulam a lógica de integração com o banco de dados. Esses módulos são responsáveis por gerenciar as operações de leitura e escrita, bem como garantir a segurança dos dados.

Futuras Integrações

  Embora atualmente não tenhamos uma API de pagamento implementada, consideramos a possibilidade de futuras integrações. A adição de uma API de pagamento permitirá processar transações de compra na loja virtual do portal de forma segura e eficiente, complementando as funcionalidades já existentes.

  Essa abordagem modular garante que o sistema possa evoluir e se adaptar às novas necessidades de forma eficiente, mantendo a segurança e a integridade dos dados.

Segurança

Medidas de Segurança Implementadas

  Para garantir a proteção e integridade dos dados armazenados em nosso site, implementamos diversas medidas de segurança. Entre essas medidas, destacam-se:

  1. Requisitos de Senha:

    • Mínimo de 20 caracteres: As senhas devem ter pelo menos 20 caracteres, proporcionando uma camada adicional de segurança contra ataques de força bruta.
    • Pelo menos 1 letra maiúscula: Incluindo letras maiúsculas, aumentamos a complexidade da senha.
    • Pelo menos 1 letra minúscula: A inclusão de letras minúsculas é fundamental para criar senhas mais seguras.
    • Pelo menos 1 número: Números aumentam a variabilidade e segurança das senhas.
    • Pelo menos 1 caractere especial: Caractere especial adiciona complexidade adicional às senhas, tornando-as mais difíceis de quebrar.

  2. Recomendações de Senha:

    • Senha longa: Recomendamos o uso de frases longas, que são mais seguras e ainda assim fáceis de lembrar.
    • Complexa: As senhas devem ser significativas apenas para o usuário, dificultando a adivinhação.
    • Prática: A senha deve ser fácil de lembrar para o usuário.
    • Impessoal: Evitar o uso de datas ou dados pessoais, que podem ser facilmente descobertos.
    • Secreta: Nunca compartilhar senhas com terceiros.

  3. Criptografia de Senhas:

    • Salt e Hash: Utilizamos um salt exclusivo para cada senha e aplicamos a função de hash SHA-256 para criptografar as senhas antes de armazená-las no banco de dados. Isso garante que as senhas sejam seguras mesmo se o banco de dados for comprometido.<>

Medidas de Segurança Implementadas

  Nossa política de segurança abrange todas as áreas críticas para proteger as informações do usuário e os dados sensíveis armazenados no sistema:

  1. Autenticação Segura:
    • Todos os usuários são autenticados usando métodos que verificam as credenciais de login contra os dados criptografados armazenados no banco de dados.

  2. Acesso Restrito:
    • O acesso aos dados do servidor e do banco de dados é restrito apenas ao pessoal autorizado. Utilizamos controles de acesso baseados em permissões para garantir que apenas indivíduos qualificados possam modificar ou visualizar dados sensíveis.

  3. Monitoramento:
    • Implementaremos sistemas de monitoramento contínuo para detectar e responder rapidamente a atividades suspeitas ou não autorizadas. Logs de auditoria serão mantidos para revisar o acesso e as alterações aos dados.

  4. Backup Regular:
    • Backups incrementais são realizados após cada alteração significativa e um backup completo semanalmente. Isso assegura que temos cópias de segurança atualizadas que podem ser restauradas em caso de falha ou perda de dados.

  5. Procedimentos de Recuperação:
    • Desenvolveremos procedimentos claros e documentados para a recuperação de dados, incluindo a restauração de backups e resposta a incidentes de segurança.

  6. Treinamento:
    • Recomendamos treinamento contínuo sobre práticas de segurança, incluindo o reconhecimento e a prevenção de ameaças cibernéticas.


  Essas políticas e medidas de segurança foram implementadas para proteger os dados dos usuários e garantir a integridade e disponibilidade dos serviços.

Testes e Qualidade

Estratégias de Teste Utilizadas

  • Testes de Funcionalidade:

      Os testes de funcionalidade foram realizados manualmente pela equipe do projeto para verificar todos os aspectos interativos do site. Isso inclui testes de links, imagens, alinhamento, botões, formulários e outras funcionalidades. Esses testes garantem que todos os elementos do site funcionem conforme o esperado e proporcionem uma experiência de usuário suave e intuitiva.

  • Testes de Integração:

      Além dos testes de funcionalidade, a equipe realizou testes de integração para verificar a interação entre diferentes componentes do sistema. Isso inclui testes de integração entre o backend e o frontend. Os testes de integração garantem que as diferentes partes do sistema se comuniquem e funcionem bem juntas, detectando problemas de comunicação, dependências e compatibilidade entre os componentes.

  • Testes de Aceitação:

      Os testes de aceitação foram conduzidos para validar se o sistema atende aos requisitos e às expectativas dos usuários finais. Esses testes, realizados de forma manual, verificam cenários de uso reais do sistema, garantindo que todas as funcionalidades estejam implementadas corretamente e que a experiência do usuário seja satisfatória.

  • Testes com Usuários:

      Para complementar os testes realizados pela equipe, foram conduzidos testes com usuários reais. Os usuários testaram o site e forneceram feedback através de questionários. Esses testes são essenciais para entender a experiência do usuário e identificar áreas que possam precisar de melhorias. O feedback dos usuários foi analisado e utilizado para fazer ajustes e aprimoramentos no sistema, garantindo que ele atenda às necessidades e expectativas do público-alvo.

  Essas estratégias de teste e qualidade garantem que o sistema Alpha Strike seja confiável e atenda aos mais altos padrões de qualidade, proporcionando uma experiência consistente e satisfatória aos usuários finais.
